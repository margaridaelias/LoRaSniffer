A Test-Repeater is a LoRaSniffer that you can connect to with another LoRaSniffer. It allowes to test LoRa radio setups over distance and can recover from lost connectivity, as long as the connection can be established on the LDC (Long Distance Channel).

The purpose of the Test-Repeater is to establish the optimal radio setup between 2 LoRa radios, in a given situation. To use this feature you need to have 1 LoRaSniffer setup as a Test-Repeater while another Sniffer works as usual. A Test-Repeater is a LoRaSniffer set into a special mode. In this mode the Sniffer will listen on the LDC for incoming radio setups messages. When it receives one:
- it will switch to this setup. - send an "Error 0: All OK!" message back to the caller. - and will stay on this setup until a switch-back to LDC is requested (the dialog is closed).
Idealy the Test-Repeater is placed where you want to setup your LoRa-network aggregator. The other LoRaSniffer you will move around to where you want to place nodes.

These are some of the standard LoRaWAN radio settings for the EU and the US. Please refer to the appropriate legislation for your area.

This allowes to measure the current overall RSSI of the environment within the current radio setup. It serves as an indicator for the overall radio noise in the area.

You can check if any preamble is received in the set time. This usally is used to implement "Listen before Transmit". This function will capture all preambles in the range of the currently setup frequency plus/minus half of the setup bandwidth. If used for instance with a bandwidth of 500 kHz, it can/will capture as well the preambles of transmissions with 250 kHz and 125 kHz bandwidth.

Here you can calculate how muc airtime a given amount of byte will need to be send using the current radio setup. The time returned is the absolute minimum and does not account for delays due to message processing.

Here you can increase or diminuish the size of the font the app uses. The default is 100%. 150% would be a font one and a half time bigger, 50 would be a font half the size.

This dialog allowes to send PINGs and receive the answer from another LoRaSniffer (or Test-Repeater). The quality of the transmission is shown in the graph, with SNR on the vertical and RSSI on the horizontal axis. The green area is a (very) good reception, the yellow one is still acceptable, but the orange and red areas should be avoided.

The LDC is the Long Distance Channel. This is the radio setup that allowes for the longest transmission distance possible. It uses the highest Spreading Factor, the lowest Bandwidth, the deepest Error Correction Rate and the highest transmission power possible to guarantee successful transmission. Of course this comes at the cost of higher battery consumption, but which in the case of the LoRaSniffer is not really an issue.
